Threads, Cores, and Goroutines Explained

March 2026 · 6 min read

Modern programs rely heavily on concurrency to fully utilize modern CPUs. Understanding threads, cores, and goroutines is essential for writing efficient software.

CPU Cores

A CPU core is a physical execution unit inside a processor.

  • Each core can execute instructions independently
  • Modern CPUs often have 4–16 cores
  • Parallel programs can use multiple cores simultaneously

Threads

Threads are lightweight units of execution within a process.

  • Managed by the operating system
  • Multiple threads share the same memory
  • Context switching between threads has overhead

Goroutines

Go introduces goroutines — extremely lightweight threads managed by the Go runtime.

  • Thousands of goroutines can run concurrently
  • Scheduled by the Go runtime instead of the OS
  • Very small memory footprint (~2 KB stack)

How Go Uses CPU Cores

The Go scheduler maps goroutines to OS threads which run on CPU cores. 

Goroutines → Go Scheduler → OS Threads → CPU Cores

Why Goroutines Are Powerful

  • Extremely lightweight
  • Easy concurrency with channels
  • Efficient scheduling

This design allows Go programs to handle massive concurrency with minimal complexity.
